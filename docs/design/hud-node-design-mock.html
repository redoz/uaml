HUD node design

A node-type design editor in the shared HUD material: frosted panel, thin source-bright frame, low glow. A live preview sits on its own inset canvas and reacts to every control. Appearance leads with the accent identity colour — the thing you change most. Header sets whether the top band shows and its treatment (Band / Fill). Body lists the compartments — Attributes, Operations, Ports — drag-reorderable by the grip, the order mirroring straight into the preview; each toggles on/off and a disabled one is simply not rendered, its own options dimming to inert (Operations, here). Under Attributes and Operations, column chips choose which member fields show: Name is locked on, then Visibility, Type, Cardinality (attributes) or Visibility, Params, Return (operations). Allowed stereotypes reuses the slant-tag editor — the stereotypes an instance may carry (the slant reads as guillemets, so the «» drop away); empty allows any. Corner radius, border weight and glow are canvas-wide Atlas theme, set once elsewhere.
