# OKF Canvas — static SPA pivot

**Date:** 2026-07-10
**Status:** Approved design, pre-implementation
**Fork of:** OWOX Model Canvas (Apache-2.0)

## Goal

Turn OWOX Model Canvas into **OKF Canvas**: a purely static single-page app that
sketches data models on a React Flow canvas and shares them via URL. Strip every
server-side and vendor-coupled component. No backend we run, no accounts, no
external tracking, no OWOX product branding.

## Non-goals

- No deploy target yet — CI produces the static `dist/` artifact; hosting is a
  later decision.
- No new features. This is a subtractive refactor plus a rebrand.
- No change to the URL share format (gzipped JSON) or the core canvas UX.

## What stays

The actual product:

- React Flow canvas — node/edge create, drag, connect, delete.
- Field-level ERD view (schema columns, PKs, join keys).
- Templates (industry + dataset starter models).
- OKF import/export via `@mc/okf` (parse/serialize markdown bundles).
- URL sharing — whole model gzip+JSON packed in the URL hash (`#m=…`),
  anonymous, never sent to any server.
- localStorage persistence (current model, name, view mode, rel labels, goal).

## What gets deleted

| Area | Files / surface |
|---|---|
| Server (whole package) | `packages/server/**`, its tests, `render.yaml` web-service |
| OWOX push | `web/src/sync/push.ts`, `PushConfirmDialog`, `OwoxImportDialog`, `Canvas.pushgate`, push-gate state |
| Supabase / accounts | `lib/supabase`, `lib/account`, `lib/auth`, `AccountPanel`, `SignInModal`, `lib/models` (cloud CRUD), version history, `supabase/**` migrations |
| AI questions | `inspector/QuestionsPanel`, `lib/questions`, all Gemini code (server-only anyway) |
| Analytics | `analytics/posthog`, PostHog script + init |
| Branding | `OwoxDataMartsHero`, OWOX product name/logos/copy across UI + README |

Roughly half of `web/src/components` goes. Remaining components rewire to the
local-only store (no auth gating, no "save to cloud", no "push").

## Format decisions (OKF)

OKF = "Open Knowledge Format", an OWOX-associated markdown format. There is no
independent standard. The vendored `test/fixtures/google/*` bundles are OKF files
describing Google BigQuery *public datasets*, generated by OWOX-ecosystem tooling
— not a Google-native format. OKF Canvas owns its own clean format.

1. **`type` is opaque.** No filtering, no inference, no branching on it. Read and
   carry it as displayable metadata; it never drives logic.
2. **Every mart doc → node.** Drop `NON_MART_TYPE` filtering, `sourceFromType`
   BigQuery mapping, and the `type: "OWOX Data Mart"` prose-skip special-case in
   `parse.ts`. (Navigation `index.md` files remain non-nodes, distinguished by
   filename, not type.)
3. **`inputSource`** is read only when a doc states it explicitly; otherwise a
   plain default. No longer inferred from `type`/tags.
4. **Export is unbranded.** `serialize.ts` emits a neutral token
   (`type: "Data Mart"`, drop `owox` tags, reword the index description). Import
   still loads legacy `OWOX Data Mart` / older files — read stays liberal.
5. **Prose-join recovery kept** (`parse.ts:79-99`). Not type-related; it recovers
   edges from joins written in prose, which is what makes "paste messy LLM output
   → import" work. It only builds graph edges.
6. **Delete** vendored `test/fixtures/google/*` and the BigQuery/type-filtering
   tests — they exercise logic being removed. Keep roundtrip, OWOX-format (as
   legacy-import coverage), prose-join, slug, and serialize tests, updated for the
   neutral export token.

**No URL-share impact:** shares are gzipped JSON `ModelGraph`; the `type` token
only lives in OKF markdown import/export, so none of this touches shared links.

## Repo structure after

- Monorepo shrinks to two packages: `web` (the SPA) + `okf` (parse/serialize).
- Root `package.json` scripts drop server (`dev`, `serve` rewritten to web only;
  `build` = okf build + web build).
- `render.yaml`: removed or converted to a static-site stub (deploy deferred).
- CI: drop the server test leg; keep lint + build + test on `web` + `okf`; keep
  ubuntu+windows matrix.

## Testing strategy

- Every deletion must leave the suite green — remove dead tests alongside dead
  code, never leave orphaned imports.
- okf: update serialize/roundtrip tests for the neutral export token; keep
  prose-join and slug tests; delete google/BigQuery-filter tests.
- web: delete tests for deleted components (push, account, auth, questions,
  signin, owox hero); the surviving canvas/import/share/template/state tests must
  still pass.
- CI (lint + build + test, ubuntu+windows) is the completion gate for each phase.

## Rollout

Subtractive refactor, staged so `main` stays green at every step:

1. Rebrand + license (name, README, NOTICE) — low risk, no logic.
2. Delete AI + analytics (self-contained).
3. Delete OWOX push flow.
4. Delete Supabase / accounts / versions; rewire canvas to local-only store.
5. Delete `packages/server`; update root scripts, `render.yaml`, CI.
6. Simplify `okf` format handling (type opaque, neutral export, drop google
   fixtures).

Each stage: change → suite green → commit → push to `main`.
